# Workflow Git recomandat

## Branch-uri

- `main` pentru versiunea stabila
- `develop` pentru integrarea curenta
- `feature/*` pentru functionalitati noi
- `fix/*` pentru bug fix-uri

## Reguli

- fiecare membru lucreaza pe branch separat
- fiecare feature important trece prin pull request
- fiecare student trebuie sa aiba minimum 5 commit-uri relevante
- merge-ul se face dupa review intern

## Exemple

- `feature/map-weather`
- `feature/ai-debate`
- `feature/auth-favorites`
- `fix/history-save-error`
